## 3. Узагальнений каркас, який повторюється в більшості текстовок

Найстабільніша структура:

- реферат;
- abstract;
- вступ;
- розділ 1:
  - 1.1 характеристика об’єкта і постановка задачі;
  - 1.2 огляд літератури / аналогів;
  - 1.3 аналіз і вибір методів, алгоритмів та засобів;
- розділ 2:
  - 2.1 архітектура;
  - 2.2 блок-схеми алгоритмів;
  - 2.3 діаграми функціонування;
  - 2.4 структура даних і база даних;
- розділ 3:
  - 3.1 діаграма класів;
  - 3.2 бібліотеки;
  - 3.3 програмні модулі;
  - 3.4 інтерфейс;
  - 3.5 альтернативи;
  - 3.6 проблеми під час розробки;
- розділ 4:
  - 4.1 інструкції;
  - 4.2 вимоги;
  - 4.3 тестування;
  - 4.4 оцінювання результатів;
- висновки;
- джерела;
- додатки.

Для `studentflow_diploma_sections.md` збережено лише обов’язкову академічну логіку нумерації 1.1–4.4, але внутрішні назви, порядок подачі матеріалу, аналоги та предметні акценти переформовано під StudentFlow. Це зроблено, щоб текст не повторював структуру жодної конкретної роботи з вибірки.

## 5. Рішення для нової текстовки StudentFlow

Для `studentflow_diploma_sections.md` прийнято такі рішення:

- залишити академічну послідовність розділів, але не копіювати назви й ритм подачі з інших текстовок;
- не повторювати Volunteer Hub / волонтерську логіку;
- у 1.2 використати аналоги саме зі сфери студентського залучення, цифрового запису позааудиторних досягнень і студентського життя закладу: Suitable, Modern Campus Involve, CampusGroups, Anthology Engage, Pathify Communities, GivePulse;
- не використовувати LMS на кшталт Moodle або Google Classroom як основні аналоги, бо вони описують навчальні курси, а не позааудиторний маршрут;
- у 2.1 додати компонентну архітектуру;
- у 2.2 додати алгоритм маршруту студента та життєвий цикл завантаження в ImageKit;
- у 2.4 додати ER-модель Supabase;
- у 3.1 додати діаграму класів;
- у 3.3 включити реальні лістинги з `repository.ts`, `services.ts`, `forms.tsx`, `supabase-wake.ts`;
- обмежити текст до 1000 рядків;
- зробити частину розділів списками, а частину — щільними абзацами.
